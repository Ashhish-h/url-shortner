# URL Shortener with Analytics

A production-grade URL shortening service built with Java Spring Boot, featuring Redis caching, click analytics, and rate limiting. Built as Project 1 of a structured backend engineering learning arc targeting SDE-1 level competency.

---

## What It Does

- Accepts a long URL and returns a short, shareable code
- Redirects short codes to their original URLs
- Tracks every click with metadata (IP, device, referrer)
- Exposes analytics — total clicks, leaderboard rank, recent click history
- Rate limits requests per IP to prevent abuse

## Architecture Overview

```
Client
  ↓
RateLimitInterceptor       (token bucket via Redis, per IP)
  ↓
Controller Layer           (thin HTTP adapter, zero business logic)
  ↓
Service Layer              (all business logic lives here)
  ↓              ↓
Redis          PostgreSQL
  ↑
AnalyticsService           (async, decoupled via Spring Events)
```

### Single Service Architecture
This is a single Spring Boot service — not microservices. Code is structured with clean separation of concerns so it could be split into microservices in a future iteration.

## Implementaton

### Base62 Encoding
Short codes are generated by Base62-encoding the auto-incremented database ID. With 6 characters, this supports ~56 billion unique URLs with zero collision risk — the database guarantees unique IDs.

```
DB ID: 10045  →  Base62 encode  →  "aB3kx9"
```

A shuffled character alphabet prevents sequential enumeration of URLs.

### Caching Strategy
| Operation | Strategy | Reason |
|---|---|---|
| Create URL | Write-around | No point caching — nobody has visited yet |
| Redirect | Cache-aside | Check Redis first, miss → DB → populate cache |
| Deactivate | Cache invalidation | Delete Redis key, repopulate on next read |

Eviction policy: `allkeys-lru` — appropriate for read-heavy workloads.

### Async Click Tracking (Observer Pattern)
Click recording is fully decoupled from the redirect response using Spring's event system:

```
UrlService.getOriginalUrl()
    → returns redirect URL immediately
    → publishEvent(ClickEvent)        // fire and forget

AnalyticsService.recordClick()        // separate thread (@Async)
    → save UrlClick to PostgreSQL     // source of truth first
    → increment Redis click counter
    → update Redis leaderboard
```

The user gets their redirect in milliseconds. Analytics processing happens in the background. DB is always written first — Redis is a fast read layer, not the source of truth.

### Token Bucket Rate Limiting
Each IP address gets a bucket of N tokens per minute stored in Redis. Each request consumes one token. When the bucket is empty, requests are rejected with `429 Too Many Requests`. The bucket refills automatically via Redis TTL.

Implemented as a `HandlerInterceptor` — runs before every request reaches the controller.

## Running Locally

### Prerequisites
- Java 17+
- PostgreSQL running on port 5432
- Redis running on port 6379

### Steps
```bash
# Clone the repository
git clone https://github.com/your-username/url-shortener.git
cd url-shortener

# Configure application.properties with your DB credentials

# Run
./mvnw spring-boot:run
```
